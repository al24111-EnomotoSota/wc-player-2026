# 要件定義書 — 2026 W杯 選手図鑑 Webアプリ

> バージョン: 0.1  
> 作成日: 2026-06-27  
> ステータス: ドラフト（レビュー待ち）

---

## 1. アプリの目的

2026 FIFA ワールドカップに出場する選手の情報を一覧・検索・閲覧できる日本語の選手図鑑サイトを構築する。

- サッカーファンが「選手名を調べたい」「この国の代表メンバーを知りたい」というニーズに応える
- 長期的にはオーガニック検索流入を積み上げ、広告収益を得ることを目指す
- ログイン不要の閲覧専用サイトとして運営コストと複雑性を最小化する

## 3. 主要機能

```
選手図鑑サイト
├── 選手一覧・検索
│   ├── 国別フィルタ
│   ├── ポジション別フィルタ
│   └── フリーワード検索（名前）
├── 選手詳細ページ
│   ├── 基本情報（名前・年齢・国籍・身長・体重）
│   ├── 所属クラブ
│   ├── ポジション
│   └── 大会成績（ゴール・アシスト数）
├── 国・代表チームページ
│   ├── チーム情報（監督・グループ）
│   └── 登録メンバー一覧
└── 大会情報ページ
    ├── グループ一覧
    └── 試合日程
```

## 4. 必須機能（MVP）

MVP = 大会開幕までにリリースすべき最小限の機能セット

| # | 機能 | 説明 |
|---|------|------|
| M1 | 選手一覧 | 全出場選手をカード形式で表示。国・ポジションでフィルタ可能 |
| M2 | 選手詳細ページ | 各選手の専用URL（SEO対応）。基本プロフィールを表示 |
| M3 | 代表チーム一覧 | 出場32カ国（予定48カ国）のチーム一覧ページ |
| M4 | 代表チーム詳細ページ | チームの基本情報とメンバーリスト |
| M5 | キーワード検索 | 選手名・国名のフリーワード検索 |
| M6 | レスポンシブデザイン | スマートフォンで快適に閲覧できる UI |
| M7 | SEO 基本対応 | ページごとの title・description・OGP タグ設定 |

## 7. 非機能要件

### 7.1 パフォーマンス

| 指標 | 目標値 |
|------|--------|
| LCP（Largest Contentful Paint） | < 2.5 秒 |
| FID / INP | < 200 ms |
| CLS | < 0.1 |
| Lighthouse スコア（モバイル） | 80 点以上 |

> W杯期間中はトラフィックが急増するため、静的生成（SSG）またはCDN配信を基本とする。

### 7.2 SEO

- 選手・チームごとに一意のURLとメタタグ
- 構造化データ（JSON-LD: Person / SportsTeam / SportsEvent）
- sitemap.xml・robots.txt の自動生成
- OGP・Twitter Card 対応（SNSシェア時にサムネイル表示）

### 7.3 セキュリティ

- HTTPS 必須
- CSP（Content Security Policy）ヘッダー設定
- 外部APIキーはサーバーサイドのみで使用（フロントへの露出禁止）
- ログイン機能なし → 認証系脆弱性のリスクなし

### 7.4 可用性

- 目標稼働率：99.5%（月間ダウンタイム < 3.6 時間）
- CDN によるエラー耐性確保

### 7.5 アクセシビリティ

- WCAG 2.1 レベル AA 準拠を目標
- 画像には alt テキスト必須
- キーボード操作対応

### 7.6 保守性

- TypeScript の使用で型安全を確保
- コンポーネント単位でのUIモジュール化
- データ（選手・チーム）をコンテンツ層として分離

## 8. 使用 API 候補

| API | 無料枠 | 主な提供データ | 注意点 |
|-----|--------|----------------|--------|
| **[API-Football](https://www.api-football.com/)** | 100 req/日 | 選手・チーム・試合・スタッツ | 商用利用は有料プラン必須（月額〜$10〜） |
| **[football-data.org](https://www.football-data.org/)** | 無料Tier あり | チーム・試合・スコア | W杯データは有料Tier の可能性あり |
| **[TheSportsDB](https://www.thesportsdb.com/)** | 無料（非商用） | 選手画像・チームロゴ・基本情報 | 商用利用はパトロン登録（$5/月〜）が必要 |
| **[OpenFootball（GitHub）](https://github.com/openfootball)** | 完全無料 | W杯の試合・スコア（静的JSON） | リアルタイムではなく事後データ |
| 独自スクレイピング | — | 任意 | Transfermarkt 等はToS 違反リスク高・推奨しない |

> **推奨方針：** MVP では OpenFootball の静的JSONで初期データを構築し、大会中はAPI-Football（有料プラン）でリアルタイム成績を更新する二段階方式を検討。

## 9. ライセンス・法的注意点

| 項目 | 注意内容 |
|------|----------|
| **選手写真** | Getty Images・Reuters 等の報道写真は商用利用不可。WikimediaCommons（CC ライセンス）または各選手の公式 SNS 画像を条件確認の上使用。最悪はイラスト・プレースホルダーで代替 |
| **FIFA ロゴ・エンブレム** | FIFA の知的財産。公式ライセンスなしに商用利用不可。各国協会の紋章も同様 |
| **API 利用規約** | 広告収益 = 商用利用とみなされる場合が多い。API ごとに「Commercial Use」条項を必ず確認 |
| **データの帰属表示** | データ提供元（API-Football、TheSportsDB 等）のクレジットを About ページと footer に記載 |
| **個人情報** | ログインなし・Cookie 最小化の方針ならば個人情報法上の制約は小さい。ただし Google AdSense 導入時は Cookie 同意バナーが必要（GDPR 対応） |
| **アフィリエイト・広告** | 景品表示法・特定商取引法の観点でステマ・誇大表現は避ける |

## 10. 技術スタック候補

### 推奨構成（長期運営・SEO・広告収益を考慮）

| 役割 | 推奨 | 理由 |
|------|------|------|
| **フレームワーク** | Next.js 15（App Router） | SSG/SSR が混在可能、SEO に強い、Vercel とのシナジー |
| **言語** | TypeScript | 長期保守性・バグ予防 |
| **スタイリング** | Tailwind CSS | 高速開発、デザイン一貫性、バンドルサイズ小 |
| **データフェッチ** | TanStack Query（React Query） | キャッシュ・再フェッチ管理が容易 |
| **データベース** | Supabase（PostgreSQL） | 無料枠あり、管理画面付き、Row Level Security |
| **ホスティング** | Vercel | Next.js と最相性。CDN・Preview Deploy 込み |
| **画像最適化** | Next.js Image / Cloudinary | 自動 WebP 変換・リサイズ |
| **分析** | Google Analytics 4 | 無料・標準的 |
| **広告** | Google AdSense | 導入最小コスト |

### 代替構成（Vue/Nuxt 派向け）

| 役割 | 代替 |
|------|------|
| フレームワーク | Nuxt 3（Vue 3） |
| スタイリング | Tailwind CSS（同じ） |
| ホスティング | Vercel または Netlify |
